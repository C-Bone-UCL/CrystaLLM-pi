# AI Usage Disclosure

This document outlines the usage of Artificial Intelligence tools in the development and maintenance of this codebase. The integration of AI is strictly limited to augmenting productivity, with all core logic, testing, and final outputs subject to my personal oversight and validation.

## 1. Tools Utilized

## 2. Scope of Assistance
AI assistance was restricted to accelerating routine tasks:
*   **Boilerplate Generation:** Scaffolding initial code structures and repetitive patterns prior to manual expansion.
*   **Documentation Refinement:** Distilling complex logic into concise, readable docstrings to ensure maintainability.
*   **Test Ideation:** Brainstorming edge cases and potential test scenarios.
*   **Code Optimization:** Identifying opportunities to reduce redundancy, help with parallelisation of code and improve syntax/variable names.

## 3. Verification and Accountability
Ultimate responsibility for the correctness, scientific validity, and stability of this project rests entirely with (me) the author. 
What I did:
*   **Manual Review & Expansion:** Every snippet of code, including AI-suggested boilerplate, is reviewed, debugged, and expanded upon manually before integration.
*   **Rigorous Testing:** All unit tests and containerized testing workflows were manually ideated and executed.
*   **Output Validation:** All system outputs have been iteratively verified multiple times to ensure absolute accuracy and expected behavior.

## 4. Strict Boundaries
To guarantee the integrity of the core implementation, the following boundaries were strictly enforced during development:
*   **No Autonomous File Modification:** Zero automated file changes or unvalidated code modifications were permitted at any stage of the workflow.
